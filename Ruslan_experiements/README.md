# Двухстадийная детекция от Руслана

## Общая архитектура

### Этап 1: Детекция людей
Ансамбль из трех детекторов с последующим WBF (Weighted Box Fusion).

#### 1. YOLOv26l
- **Параметры**: imgsz=1024, batch=32, epochs=60, cos_lr=True
- **Аугментация**: 
  - HSV: h=0.005, s=0.25, v=0.25
  - Геометрические: degrees=5.0, translate=0.15, scale=0.5
  - fliplr=0.5, mosaic=1.0, close_mosaic=10
- **Результат**: mAP50-95 = 0.710

##### Fine-tuning с дополнительными данными
- Заморожены первые 150 слоев
- 10 эпох, lr0=0.002, warmup_epochs=2
- **Аугментация** (уменьшена):
  - degrees=1.0, translate=0.1, scale=0.3
  - mosaic=0.5, close_mosaic=5
- **Результат**: mAP50-95 = 0.734

#### 2. RT-DETR-l (Обучил коллега)
- **Параметры**: imgsz=640
- **Результат**: mAP50-95 = 0.730

#### 3. YOLOv8m (с SAHI)
- **Подготовка данных**: скрипт `src_split_dataset_for_SAHI.py` (слайсинг изображений 640x640)
- **Параметры**: imgsz=640, batch=32, epochs=50, cos_lr=True
- **Аугментация**:
  - HSV: h=0.001, s=0.25, v=0.25
  - Геометрические: degrees=3.0, translate=0.25, scale=0.65
  - fliplr=0.5, mosaic=1.0, close_mosaic=10
  - mixup=0.2, cutmix=0.25, erasing=0.1
- **Результат**: mAP50-95 = 0.710

#### Weighted Box Fusion (WBF)
- **Веса**: mAP50-95 каждой модели
- **Настройка**: параметры iou_thr и skip_box_thr подбирались отдельно
- **Результат**: объединенные предсказания всех детекторов

#### Кроппинг
После WBF изображения людей вырезаются с коэффициентом расширения (expand_factor), который настраивается вручную.

### Этап 2: Классификация (покупатель/сотрудник)

Используется ансамбль из 4 классификаторов, разделенных на две группы.

#### Подготовка данных
- **Группа 1**: бинарная классификация (покупатель, сотрудник), expand_factor=1.1
- **Группа 2**: 3 класса (покупатель, сотрудник, другое)
  - Для первых двух классов: expand_factor=random.uniform(1.05, 1.10)
  - Класс "другое": создан вручную скриптом `scr_create_other.py`

#### Группа 1 (бинарная классификация)

**Модель 1: tf_efficientnetv2_s.in21k_ft_in1k**
- F1-score: 0.9757
- Параметры:
  - CrossEntropyLoss с обратными весами и label smoothing
  - AdamW с weight_decay
  - drop_rate

**Модель 2: convnext_tiny.fb_in22k_ft_in1k**
- F1-score: 0.9723
- Параметры:
  - SoftTargetCrossEntropy
  - Семплирование
  - AdamW с weight_decay
  - drop_rate, Mixup

#### Группа 2 (3 класса)

**Модель 1: tf_efficientnetv2_s.in21k_ft_in1k**
- F1-score: 0.9275
- Аналогичные параметры

**Модель 2: convnext_tiny.fb_in22k_ft_in1k**
- F1-score: 0.9378
- Аналогичные параметры

## Скрипты

### `src_split_dataset.py`
Разделение датасета 80/20 с учетом источника данных (по камерам). Гарантируя, что кадры с каждой камеры равномерно распределены между train и val.

### `src_split_dataset_for_SAHI.py`
Подготовка данных для SAHI: нарезка изображений 1280x720 на слайсы 640x640. Разбиение 80/20 выполняется с учетом количества людей на изображениях.

### `src_crop_people.py`
Создание датасета для классификаторов: вырезание людей с изображений по заданным координатам.

### `scr_create_other.py`
Вспомогательный скрипт для ручного создания изображений класса "другое".
