# Best Model — Описание директории

Данная директория содержит скрипты и ноутбуки для обучения ансамбля моделей детекции объектов (YOLO и RT-DETR) с использованием K-Fold кросс-валидации.

---

## Структура файлов

| Файл | Описание |
|------|----------|
| `create_fold.py` | Скрипт для создания K-Fold разбиений датасета с аугментацией |
| `train_model.ipynb` | Jupyter-ноутбук для обучения моделей YOLO и RT-DETR на всех фолдах |
| `predict_with_model.py` | Скрипт для инференса обученных моделей на тестовых изображениях |
| `sub_create_wbf.py` | Скрипт для объединения предсказаний с помощью Weighted Boxes Fusion (WBF) |
| `README.md` | Этот файл |

---

## Описание файлов

### 1. `create_fold.py`

**Назначение:** Генерация 5 фолдов для кросс-валидации с аугментацией данных.

**Что делает:**
- Собирает все изображения из папок `train` и `val` исходного датасета
- Создаёт 5 фолдов с помощью `KFold` из `sklearn`
- Для **train-части** каждого фолда создаёт дополнительную аугментированную копию каждого изображения через **случайный кроп** (60–90% от оригинала) с пересчётом bounding boxes
- Для **validation-части** сохраняет только оригинальные изображения (без аугментации)
- Сохраняет результат в папку `dataset_folds/fold_{N}` с структурой YOLO

**Настройки:**
- `NUM_FOLDS = 5` — количество фолдов
- `CROP_SCALE_MIN = 0.6`, `CROP_SCALE_MAX = 0.9` — диапазон размера кропа

---

### 2. `train_model.ipynb`

**Назначение:** Обучение ансамбля моделей YOLO11m и RT-DETR-L на всех фолдах.

**Что делает:**
- Последовательно обучает модели на каждом из 5 фолдов
- Для каждой модели использует:
  - **Early Stopping** (patience=15)
  - **Аугментацию**: mosaic, mixup, copy_paste, HSV, вращение, сдвиг, масштаб
  - **Cosine Learning Rate** scheduler
  - **Gradient Clipping** для RT-DETR (кастомный optimizer_step)
  - **Агрессивную очистку памяти** после каждого фолда

**Параметры обучения:**
| Модель | Эпохи | Batch Size | Image Size |
|--------|-------|------------|------------|
| YOLO11m | 70 | 8 | 640 |
| RT-DETR-L | 100 | 8 | 640 |

---

### 3. `predict_with_model.py`

**Назначение:** Инференс всех обученных моделей на тестовых изображениях.

**Что делает:**
- Для каждого фолда загружает веса YOLO11m и RT-DETR
- Запускает предсказание на изображениях из `test_images/test_images`
- Сохраняет результаты в формате YOLO: `class_id x_center y_center width height confidence`
- Использует `augment=True` для TTA (Test Time Augmentation)

**Настройки:**
- `NUM_FOLDS = 5`
- `CONF_THRESHOLD = 0.25`
- `NUM_CLASSES = 2`

**Выход:** Папки `predictions/{model}_fold_{N}` с `.txt` файлами для каждого изображения

---

### 4. `sub_create_wbf.py`

**Назначение:** Объединение предсказаний всех моделей с помощью **Weighted Boxes Fusion (WBF)** и создание финальной submission-таблицы.

**Что делает:**
- Читает предсказания всех моделей для каждого изображения
- Конвертирует боксы из формата YOLO в XYXY для WBF
- Применяет WBF с весами моделей и порогом IoU
- Фильтрует боксы по финальному порогу уверенности
- Конвертирует обратно в YOLO-формат и сохраняет в CSV

**Настройки WBF:**
- `MODEL_WEIGHTS = [1.1, 1.1, 1.0, 1.0]` — веса для моделей
- `IOU_THR = 0.5` — порог пересечения для слияния боксов
- `SKIP_BOX_THR = 0.001` — игнорировать боксы с confidence ниже этого до WBF
- `CONF_THR_FINAL = 0.25` — финальный порог после WBF
- `keep_only_class = 1` — опционально: оставить только определённый класс

**Выход:** `submission_wbf.csv` — готовый файл для отправки
